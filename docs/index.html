// Seams

6 UV seam algorithms for any mesh type. Select a mesh, choose a method, unwrap.

Methods

Seams by Angle

Marks edges as seams where the dihedral angle between adjacent faces exceeds a threshold. General-purpose, good first pass for any model.

ParameterDefaultDescription
Angle Threshold30°0–180°. Lower = more seams
Clear ExistingtrueRemove existing seams first
Seams by Material

Marks edges where adjacent faces have different material assignments. Best for models with distinct material regions (skin vs. clothing, metal vs. rubber).

Seams by Hard Edge

Marks seams at sharp/hard edges using Blender 5.0's sharp_edge attribute. Best for hard-surface models where you've already marked sharp edges for shading.

Island-Aware Seams

Iterative algorithm: starts with angle-based seams, then adds more at high-distortion edges. Control over UV island count and stretch.

ParameterDefaultDescription
Max Islands0 (unlimited)Cap on island count
Max Stretch0.50–1 target distortion threshold
Seams by Projection

Projection-based seams for simple shapes. Modes:

  • BOX — Seams where face normals change dominant axis
  • CYLINDER — Seams at cap transitions + one vertical cut
  • SPHERE — Single longitude cut
Clear All Seams

Removes all seams from the active mesh. Useful before applying a new method from scratch.

Presets

PresetMethods

hard_surfaceHard Edge + Angle (35°) + Material
organicIsland-Aware + Material

// Rigging

13 modules, skeleton scanner, wrap rig. Universal CTRL → MCH → DEF architecture across everything.

Three-Tier Architecture
DEF (deform): only COPY_TRANSFORMS from MCH. Never carries rig logic. use_deform=True.
MCH (mechanism): all rig logic (FK, IK, DAMPED_TRACK, STRETCH_TO). use_deform=False.
CTRL (controls): user-facing bones for animation. use_deform=False.
Rig logic never touches deform bones directly.

FK/IK System

FK/IK Toggle & Snapping

Works on any chain with IK constraints. All chains start in FK mode.

ModeMCH FK ConstraintMCH IK Constraint
FKinfluence = 1.0influence = 0.0
IKinfluence = 0.0influence = 1.0

IK range-aware: bones outside the IK chain_count (e.g. foot/toe below IK target) keep FK active in both modes.

Snapping (Standard IK, arms/legs)

  • IK → FK: FK bones snap to match the IK-solved pose
  • FK → IK: IK target snaps to chain end, pole angle recalibrated

Snapping (Spline IK, tails/tentacles)

  • Spline → FK: FK controls snap to spline-solved MCH poses
  • FK → Spline: Hook bones repositioned along FK-posed chain

IK visibility: IK target/pole/hook bones are in per-chain collections (IK_arm_L, IK_tail_C) that show/hide with FK/IK toggle.

python blender_api.py rig-toggle-fk-ik --armature Armature --chain leg_L --mode IK
python blender_api.py rig-toggle-fk-ik --armature Armature --chain arm_R --mode TOGGLE
Spline IK (Tails, Tentacles)

For long chains, standard IK distributes motion poorly. Spline IK uses a Bezier curve driven by hook control bones for smooth deformation.

  1. Bezier curve BT_Wrap_Spline_{chain_id} created along chain, parented to armature
  2. 3–5 hook bones (CTRL-Wrap_{chain}_Spline_00 etc.) at even intervals
  3. Hook modifiers bind curve control points to hook bones
  4. SPLINE_IK on the last MCH bone drives the whole chain
  5. Moving hook bones reshapes the curve, smoothly deforming all bones

Control points: 3 for ≤4 bones, 4 for ≤8, 5 for 9+. All hooks parented to cross-chain MCH parent. Defaults to SPLINE for tail/tentacle.

IK Rotation Limits

Uses Blender's IK bone limit properties (ik_min_x, ik_max_x, ik_stiffness_x) which are evaluated inside the IK solver — unlike LIMIT_ROTATION constraints which IK ignores.

ModuleDefaults
Arm/Leg mid-jointsSingle-axis bend (auto-detected), 0°–160°, secondary axes locked (stiffness 0.9)
Arm/Leg root/end±120° all axes
Tail±45°, stiffness 0.1
Tentacle±60°, stiffness 0.05
Generic±90°, no stiffness

Runtime toggle: bt.toggle_ik_limits flips limits on/off per chain. Values preserved.
Per-bone editor: bt.edit_bone_ik_limits (gear icon or right-click → "Edit IK Limits").

FK Location Locks New

FK control bones have their location channels locked (lock_location = (True, True, True)) so they can only be rotated, not translated. This enforces proper FK workflow where rotation drives the chain.

COG Exception

The first FK bone of spine/root chains (Center of Gravity bone) keeps location unlocked. This allows the COG bone to translate the entire body while all other FK bones below it rotate only.

  • Applied automatically during wrap rig generation via assign_shapes_for_wrap_rig()
  • Also applied during modular rig generation
  • COG bones are identified as the first FK bone in the chain hierarchy
Why lock FK location?
Translating FK bones breaks the chain hierarchy and produces unexpected deformation. Locking location ensures animators use FK bones as intended: rotate joints, translate only the COG for overall body movement.
Floor Contact

Prevents IK feet from going below a configurable floor level. Optional auto toe bend on ground contact.

  • Floor constraint: LIMIT_LOCATION (min Z, world space) on each leg IK target
  • Toe auto-bend: TRANSFORM constraint maps IK target Z to toe X rotation (active in IK mode only)
python blender_api.py floor-contact --armature Armature --action enable --level 0
python blender_api.py floor-contact --armature Armature --action disable

Clickable bar at the viewport bottom showing FK/IK state per chain. Blue = FK, Orange = IK. Click to toggle. Shows "Spline" for spline IK chains.

Center of Mass + Balance

Pose-mode visualization for balance analysis.

Center of Mass (CoM)

  • Crosshair at weighted average of deform bone positions
  • Mass auto-calculated from mesh vertex weights; per-bone override available
  • Colored by balance: green (stable) → yellow → red (unstable)

Base of Support (BoS)

  • Convex hull polygon from foot/hand contact bones projected on ground
  • Detection tiers: scan data → name matching (foot/toe/hand/paw/hoof/claw) → Z-position (leaf deform bones near ground)
  • Works for bipeds and quadrupeds

Balance Indicator

  • Stability ratio bar at bottom-right with needle
  • Green (>50%) → yellow (50%) → red (0% = on edge)

// Animation

Smart Keyframe System New

Overrides the I key in Pose mode via addon keymap. Instead of Blender's default Insert Keyframe behavior, Smart Keyframe intelligently determines what to key and how based on bone type and chain state.

Core Principle
Never key IK bones directly. If an IK bone is selected and the chain is in IK mode, Smart Keyframe snaps FK to match the current IK pose, then keys the FK bone rotations instead. This ensures animation data lives on FK bones and survives FK/IK mode switches.
Bone TypeWhat Gets KeyedDetails
IK bone (chain in IK mode)FK rotationSnaps FK to match IK pose first, then keys FK bone rotations
FK boneRotationKeys rotation channels directly
COG / root boneLocation + RotationCOG bones have unlocked location, so both channels are keyed
Non-wrap boneRotation + Location (if unlocked)Falls back to standard keying for bones outside the wrap rig

Operator: bt.smart_keyframe

Workflow Benefit
Animate freely in IK mode (drag IK targets for posing), press I to key, and the FK bones receive the keyframes. Switch to FK mode at any time — all animation is preserved on the FK controls.

Bone Trajectory Updated

Interactive 3D visualization of bone motion paths across keyframes. Select bones in Pose mode, enable via header Overlays or Animation > Trajectory panel.

Universal Trajectory
Trajectory now works for all bone types, not just location-keyed bones. Any keyframe type (rotation, location, scale) is used for visualization. The trajectory curve shows the bone's world-space position at each keyed frame regardless of which channels are keyed.

Dot Colors

ColorMeaningInteraction
YellowDirectly editableBone has location keyframes. Click + drag to edit position. World-space delta is inverse-transformed to bone-local coords and FCurve values update in real-time.
CyanIK-assisted editingFK bone with IK available on its chain. Click to engage temporary IK workflow (see below).

IK-Assisted Editing (Cyan Dots)

When you click a cyan dot, the following sequence occurs:

  1. IK target snaps to match the current FK pose
  2. Chain temporarily switches to IK mode
  3. Drag the IK target to reposition the chain end
  4. On mouse release: FK bones snap to match the new IK-solved pose
  5. FK rotation keyframes are inserted for all chain bones at that frame
  6. Chain constraints restored to original FK/IK state

The viewport header displays a hint showing available actions based on the dot types visible in the current selection.

Trajectory curve colors: past = blue, future = green.

Operator: bt.trajectory (modal, ESC to exit)

Onion Skinning Updated

Camera-independent ghost frame display for armatures with child meshes. Past ghosts: blue, future ghosts: orange, opacity fades with distance from current frame.

Settings & Properties

All settings are always visible in the Animation > Onion Skin panel (not gated behind enable state).

PropertyTypeDefaultDescription
bt_onion_beforeint3Number of ghost frames before current
bt_onion_afterint3Number of ghost frames after current
bt_onion_stepint1Frame interval between ghosts (ignored in Keyframes Only mode)
bt_onion_opacityfloat—Base opacity for nearest ghost (fades with distance)
bt_onion_use_keyframesboolfalseNew Show ghosts only at keyed frames instead of fixed intervals
bt_onion_proxy_ratiofloat0.25New Proxy LOD decimation ratio (0.05 – 1.0)
Keyframes Only Mode New

When bt_onion_use_keyframes is enabled, ghosts appear only at frames that have actual keyframes rather than at fixed step intervals. This is useful for reviewing specific poses and transitions without cluttering the viewport with in-between frames.

  • The bt_onion_before / bt_onion_after counts still apply (limiting how many keyed ghosts to show)
  • The bt_onion_step setting is ignored in this mode
  • Keyframes are detected from the active action's FCurves
Proxy LOD System New

Ghost evaluation can be expensive for high-poly meshes. The Proxy LOD system creates decimated proxy meshes for ghost rendering, dramatically improving performance.

How It Works

  1. On first enable (or ratio change), BlenderTools creates decimated copies of each armature child mesh
  2. Uses Blender's Decimate modifier in COLLAPSE mode, which preserves vertex groups for correct skeletal deformation
  3. A single depsgraph update evaluates all proxies, then bakes the decimated geometry
  4. Ghost frames are rendered from the proxy meshes instead of full-resolution originals

Settings

  • bt_onion_proxy_ratio: 0.05 (5% of original) to 1.0 (full resolution). Default 0.25 (25%)
  • Lower values = faster but coarser ghosts. 0.25 is a good balance for most characters
  • Shows a progress bar and WAIT cursor during proxy creation
Performance
At 0.25 ratio, a 50k-poly character mesh produces ~12.5k-poly ghost proxies. With 6 ghost frames, this saves evaluating ~225k triangles per frame change.
Technical Details
  • Evaluates child meshes at past/future frames via depsgraph
  • GPU-cached batches in world space — survives viewport rotation, rebuilds only on frame change
  • Proxy meshes are temporary and cleaned up on disable

Operators: bt.onion_skin (toggle), bt.onion_skin_refresh

arm = bpy.data.objects.get('Armature')
result = json.dumps({
    'bones': len(arm.data.bones),
    'names': [b.name for b in arm.data.bones]
})
"

Response Format

// Success
{"success": true, "seams_marked": 42}

// Error
{"success": false, "error": "Object not found: Cube"}
// Rig Config Format

Schema

{
  "name": "Display name",
  "modules": [
    {
      "type": "spine",
      "name": "Spine",
      "side": "C",
      "parent_bone": "",
      "position": [0, 0, 1.0],
      "options": { "bone_count": 4 }
    }
  ],
  "global_options": {
    "deform_prefix": "DEF-",
    "control_prefix": "CTRL-",
    "mechanism_prefix": "MCH-"
  }
}

Module Options

All Module Options

spine

OptionTypeDefaultDesc
bone_countint4Spine segments
ik_splineboolfalseAdd IK spline controls

arm

OptionTypeDefaultDesc
twist_bonesint1Upper arm twist count
claviclebooltrueInclude clavicle

leg

OptionTypeDefaultDesc
twist_bonesint1Thigh twist count
foot_rollbooltrueAdd foot roll

neck_head

OptionTypeDefaultDesc
neck_bonesint2Neck segments

tail

OptionTypeDefaultDesc
bone_countint6Segments
ik_tipboolfalseIK at tip
direction[x,y,z][0,1,-0.2]Growth dir

wing

OptionTypeDefaultDesc
segmentsint3Wing segments
feather_bonesint3Feathers per segment

tentacle

OptionTypeDefaultDesc
bone_countint8Segments
taperbooltrueTaper lengths

finger_chain

OptionTypeDefaultDesc
bone_countint3Phalanx count
finger_namestring"Index"Identifier

piston

OptionTypeDefaultDesc
stroke_lengthfloat0.3Stroke distance
stretch_modestring"none"none / cylinder / rod / both
bone_mappingobject{}{"Cylinder":"...", "Rod":"...", "Rod_Parent":"..."}

wheel

OptionTypeDefaultDesc
radiusfloat0.3Wheel radius
bone_mappingobject{}{"Axle":"...", "Wheel":"..."}

jaw

OptionTypeDefaultDesc
upper_jawboolfalseInclude upper jaw

eye

OptionTypeDefaultDesc
eyelidsbooltrueInclude eyelid bones
target_distancefloat1.0Aim target distance

custom_chain

OptionTypeDefaultDesc
bone_countint4Chain length
ikboolfalseAdd IK target
direction[x,y,z][0,0,1]Chain direction
